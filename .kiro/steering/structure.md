# Project Structure

## Current Organization

The project is in initial setup phase. Directory structure will be defined as the project develops.

## Folder Conventions

Standard conventions to follow:

- Keep source code organized by feature or module
- Separate configuration files from source code
- Use clear, descriptive folder names
- Group related files together

## File Naming

- Use consistent naming conventions throughout the project
- Choose either camelCase, PascalCase, or snake_case and stick with it
- Name files descriptively based on their purpose

## Configuration

Configuration files should be placed at the project root or in a dedicated config directory.
